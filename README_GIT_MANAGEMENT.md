# Git 管理ルール・運用ガイド

## 1. `.gitignore` の設定
- 依存ライブラリやビルド成果物、秘密情報（.envなど）は `.gitignore` に必ず記載しコミット対象外にする。
- IDEやOS固有ファイルも除外してリポジトリの肥大化を防ぐ。

## 2. テストコードの管理
- `tests/` ディレクトリは必ずリポジトリに含めること。
- テストはコードの品質保証に不可欠。常に最新状態を保ちましょう。

## 3. コミットのルール
- 変更は意味のある小さな単位でコミットする。
- コミットメッセージは具体的かつわかりやすく書く。
- 変更後はテストを実行し、問題なければコミットする。

## 4. ブランチ運用
- `main`（または`master`）は常に安定したコードのみ置く。
- 機能追加・修正は `feature/` や `fix/` の名前で新規ブランチを作る。
- ブランチは目的がわかる名前を付ける。

## 5. 自動テスト（CI/CD）
- プルリクエスト時に自動でテストが走る仕組みを導入する。
- テストが通るまでマージしないルールを徹底する。

## 6. ファイル削除の対策
- 削除は慎重に行い、Gitで復元可能なことを意識する。
- なるべくIDEのファイル管理機能を使い、慌ててCLIで削除しない。

## 7. コミット頻度
- こまめにコミットし、小さな作業単位で管理する。

## 8. コードレビュー
- チームがある場合は必ずコードレビューを行う。

## 9. ドキュメント管理
- READMEや設計メモなども一緒にリポジトリに含めて管理する。

---

このルールを守ることで、安定して効率的に開発が行えます。
困った時はGitの履歴やCI/CDで状況を確認しましょう。
